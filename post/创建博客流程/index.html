创建个人博客流程

参考视频连接：https://www.bilibili.com/video/BV1q4411i7gL

需要准备环境： git， VisualCode

1.下载Hugo(当前使用0.79.0)

https://github.com/gohugoio/hugo/releases

下载完后解压，然后在目录下面操作cmd

hugo

2.创建个人博客

hugo new site [你博客的名称]

3. 下载并设置主题(本人用m10c)

https://themes.gohugo.io/

#进入到你的博客目录下
cd [path]
#git 克隆主题 例如 ：
git clone https://github.com/vaga/hugo-theme-m10c.git themes/m10c

4.在本地启动个人博客

hugo server -t m10c --buildDrafts

5.开始写博客

hugo new post/blog.md
#他会在这里创建  myblog/content/post/blog.md

然后用VisualCode来写

写完之后回到根目录【myblog】

6.将个人博客部署到远程服务端

博客是由github pages提供的服务，也就是可以在github pages上建立你自己的博客。 在github上创建仓库lokitfung.github.io （用户名要和你的username一样）

#在命令行上 --thmee声明你应用的主题 --baseUrl就是你github上仓库的地址 
#--buildDrafts意为“编译草稿（draft: true的Markdown文件）”
hugo --theme=m10c --baseUrl="https://lokitfung.github.io/" --buildDrafts
#完了他会在当前目录下生成一个public文件

在生成的public上用git命令推到github上

git init
#.是全部文件的意思
git add .  

git commit -m "我的 hugo 博客第一次提交"

# 将本地的git与远程进行关联
git remote add origin https://github.com/lokitfung/lokitfung.github.io.git

#往github上推
git push -u origin master

最后访问

lokitfung.githun.io

Hugo中Markdown插入图片路径问题

解决这个问题，主要是两个方面：

  1. 在config.toml中配置baseurl。作用是无论在本地还是发布出去，图片的路径都是相对于baseurl而言的。

  2. 在Markdown图片路径中，以static为根目录写全路径。

    示例：假设测试baseurl为http://localhost:1313/，图片位置：/static/img/hugo/123.png，那么在Markdown中写作![图片说明](/img/hugo/123.png)。这时，在浏览器就可以看到图片了
